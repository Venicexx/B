# 贪吃蛇小游戏 — 设计文档

## 概述

一个 HTML 版贪吃蛇小游戏，单文件、零依赖，双击即可游玩。童趣可爱视觉风格，支持桌面键盘和移动端触控操作。

## 技术选型

| 项目 | 选择 |
|:----:|:----:|
| 文件结构 | 单文件 `snake.html`（CSS/JS 全内嵌） |
| 渲染方式 | Canvas API |
| 依赖 | 无 |
| 运行方式 | 双击浏览器打开 |

## 架构设计

### 游戏循环（Game Loop）

核心时钟驱动，每 tick 执行一次状态更新 → 渲染。

```
setInterval(gameLoop, interval)
  │
  ├─ 1. update() — 移动蛇、检测碰撞、检测食物
  └─ 2. draw()   — 清空画布 → 画网格 → 画食物 → 画蛇 → 画 HUD
```

### 数据模型

```javascript
state = {
  status: 'waiting' | 'playing' | 'paused' | 'gameover',
  snake: [{x, y}, ...],    // 蛇身坐标数组，索引 0 = 蛇头
  direction: {dx, dy},     // 当前移动方向
  nextDirection: {dx, dy}, // 下次 tick 实际方向（防一帧内多次转向）
  food: {x, y},            // 食物位置
  score: 0,                // 当前得分
  highScore: 0,            // 最高分（localStorage）
  speed: 150,              // 当前间隔（毫秒）
  tickCount: 0,            // 吃食物计数（用于换色和加速）
  grid: 20,                // 20×20 网格
  cellSize: 20,            // 每格 20px（400/20）
}
```

### 核心逻辑

| 功能 | 实现 |
|:----:|------|
| **蛇移动** | 头部插入新坐标 `unshift`，没吃到食物则 `pop` 尾部 |
| **方向控制** | 不能 180° 掉头，`nextDirection` 做一帧缓冲防误操作 |
| **食物生成** | 随机选择不在蛇身上的格子 |
| **碰撞检测** | 撞墙壁 (x,y 出界) + 撞自身 (头部与身体重合) → gameover |
| **计分** | 每吃一个 +10 分，`localStorage` 保存最高分 |
| **速度渐变** | 每吃 5 个食物，`speed -= 15`，最低 60ms |
| **暂停** | P 键/暂停按钮切换 `status`，暂停时冻结计时器 |
| **重新开始** | 重置全部状态，读档最高分 |

### 输入处理

| 输入方式 | 绑定 |
|:---------:|:----:|
| 键盘 ↑↓←→ / WASD | keydown 事件 |
| 键盘 P / Space | 暂停切换 |
| 键盘 Enter / Space | 游戏结束时可重新开始 |
| 触控滑动 | touchstart 记录起点，touchmove 计算方向 |
| 浮动方向按钮 | onclick 触发方向变更 |

## 错误 & 边界处理

- **窗口缩放**：Canvas 保持 400×400 固定大小，通过 CSS `max-width: 100%` 适配小屏
- **快速按键**：`nextDirection` 缓冲机制，同一 tick 内多次按键只取最后一次有效方向
- **食物生成覆盖**：如果蛇几乎填满网格（只剩 1 格），确保食物一定能生成
- **localStorage 不可用**：try-catch 包裹，降级为内存记分
- **触控与键盘冲突**：两者独立运行，互不干扰
