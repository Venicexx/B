# 家庭出游提醒优化设计

> 日期：2026-06-24
> 项目：私人育婴助手

## 概述

对现有月度出游推荐推送进行实用性升级，扩展推送内容维度，使每条推荐包含天气适配、适幼度评估、行程安排（含当地美食推荐）、出行清单和周边配套信息。

## 现状

- 推送频率：每月第一个周五 09:00（北京时间）
- 数据源：`data/next_trip.json`
- 脚本：`scf/trip_reminder.py`
- 推送方式：企微群机器人
- 当前内容结构简单，仅含文字推荐消息列表

## 需求

1. **天气适配** — 推送包含目的地当天天气预报及应对建议
2. **适幼度评估** — 评分 + 具体说明（推车/腰凳、母婴设施、适合月龄等）
3. **行程安排** — 整日时间线，标注每餐当地特色美食推荐
4. **出行清单** — 按季节/天气自动匹配的携带物品清单
5. **周边配套** — 餐饮、停车、卫生间等实用信息
6. **合并推送** — 每个目的地所有信息合并为一条消息
7. **主推荐 + 备选** — 分两条独立消息推送

## 推送消息格式

每条目的地独立推送，包含以下区块，各块用空行分隔：

```
🚗 7月出游推荐 · 主推荐
━━━━━━━━━━━━━━━━━━━━

[标题 + 地点 + 距离 + 推荐日期]

🌟 推荐理由
[一段概述]

🌤️ 天气适配
[天气预报 + 应对建议]

👶 适幼度：[评分]
[要点列表]

📋 行程安排
[时间线，含美食推荐]

🎒 出行清单
[物品列表]

🍜 周边配套
[配套信息列表]
━━━━━━━━━━━━━━━━━━━━
```

## 脚本逻辑

`scf/trip_reminder.py` 修改要点：

1. 读取 `data/next_trip.json` 中的 `recommendations` 列表
2. 逐条渲染为上述格式文本
3. 每条目的地独立推送一条消息
4. 推送成功后设置 `pushed: true` 标记，避免重复推送
5. 移除月份不匹配的跳过逻辑（由 workflow 的 cron 控制）

## 不变项

- 推送频率：每月第一个周五 09:00
- 推送渠道：企微群机器人（同一 webhook）
- 数据手动填写（由 Claude 在确认后更新）
- `trip-reminder.yml` workflow 配置不变
