## 技术架构

### GitHub Actions 自动推送

项目通过 **GitHub Actions** 定时推送，免费、国内可访问，无需腾讯云 SCF。

| Workflow | 触发时间 | 执行脚本 |
|----------|----------|----------|
| `科捷/.github/workflows/weather-reminder.yml` | 每天 07:30 | `科捷/scf/index.py` |
| `科捷/.github/workflows/vaccine-reminder.yml` | 每周一 08:00 | `科捷/scf/vaccine_reminder.py` |
| `科捷/.github/workflows/important-dates-reminder.yml` | 每周一 12:00 | `科捷/scf/important_dates_reminder.py` |

**架构链路：**
```
GitHub Actions（Cron 定时触发，ubuntu-latest）
  ├─ weather-reminder.yml           → scf/index.py                 → 企微 Webhook A
  ├─ vaccine-reminder.yml           → scf/vaccine_reminder.py      → 企微 Webhook B
  └─ important-dates-reminder.yml   → scf/important_dates_reminder.py  → 企微 Webhook B
```

> ⚠️ **时区注意**：GitHub Actions 跑在 UTC，脚本中已用 `today_beijing()` 替代 `date.today()` 获取北京时间。

### 手动推送（回滚通道）

```bash
# 天气 + 节日提醒
cd "E:\Claude Code(cursor)\科捷"
python scf/index.py

# 疫苗提醒
python scf/vaccine_reminder.py

# 家庭重要日期提醒（可用 --dry-run 预览不推送）
python scf/important_dates_reminder.py
```

### ⚠️ 疫苗数据同步

疫苗时间表**同时存在于两个地方**，修改时必须两边一起改：

1. **`私人/CLAUDE.md`** — 给 Claude 看的疫苗表格（人类可读）
2. **`科捷/scf/vaccine_reminder.py`** 的 `SCHEDULE` 列表 — 给 GitHub Actions 用的数据源

修改流程：当用户说"某疫苗已接种"时 → 更新 CLAUDE.md 表格状态 → 同步更新 `科捷/scf/vaccine_reminder.py` 中对应项的 `done` 字段为 `True` → git push 即生效。

### ⚠️ 凭据安全

企微 Webhook URL 硬编码在脚本中，**不要提交到公共 Git 仓库**：

| 位置 | 凭据 |
|------|------|
| `科捷/scf/index.py:16` | 企微 Webhook（天气+节日推送） |
| `科捷/scf/vaccine_reminder.py:13` | 企微 Webhook（疫苗推送） |

> 建议后续迁移到 GitHub Secrets，在 workflow 中用 `${{ secrets.WECHAT_WEBHOOK }}` 注入。
